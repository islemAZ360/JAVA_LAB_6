## 🌐 Обзор проекта

**Java Lab 6** — это рефакторинг лабораторной работы №5, в котором монолитное приложение было разделено на две независимые части: **сервер** и **клиент**, взаимодействующие по сети.

| Компонент | Роль |
|-----------|------|
| **Сервер** | Принимает входящие запросы, обрабатывает их и возвращает ответ клиенту |
| **Клиент** | Считывает команды пользователя из консоли, формирует запросы и отправляет их на сервер |

---

## 🏗 Архитектура

Проект разделён на **3 пакета (модуля)**:

```
Lab6/
├── src/
│   ├── server/       # Серверная логика (NIO, обработка команд)
│   ├── common/       # Общие классы: модели, команды, протокол
│   └── client/       # Клиентская логика (BIO, консольный интерфейс)
├── lib/              # Внешние зависимости (JAR-файлы)
└── build/            # Скомпилированные классы
```

### Схема взаимодействия

```
[Пользователь] ──ввод──▶ [Client / BIO]
                                │
                          TCP-соединение
                                │
                         [Server / NIO]
                                │
                    [Обработка + формирование Response]
                                │
                          TCP-соединение
                                │
                     [Client] ──▶ [Вывод результата]
```

## ✨ Основные возможности

### Серверная часть
- Реализована на **Java NIO (Non-Blocking I/O)** для эффективной обработки множества подключений
- Все команды из Lab 5 адаптированы под возврат унифицированного типа `Response`
- При достижении лимита подключений (`max connection`) сервер отклоняет новые каналы и уведомляет клиента
- Интегрировано логирование через **Apache Log4j 2**:
  ```java
  import org.apache.logging.log4j.LogManager;
  import org.apache.logging.log4j.Logger;
  ```

### Клиентская часть
- Реализована на **Java BIO (Blocking I/O)**
- Улучшенный консольный интерфейс (CLI/UX)
- При получении уведомления о превышении лимита подключений клиент автоматически повторяет попытку подключения (`retry`)
- При достижении максимального числа повторных попыток (`max retries`) соединение завершается

### Общая функциональность
- Рекурсивная и перекрёстная обработка `run_script_file` с защитой от бесконечной рекурсии
- Добавлены флаги для команд, требующих подтверждения (например, `clear`)
- Расширенный набор `StatusCode` enum для явного отображения статуса выполнения команд

## 🚀 Сборка и запуск

### Предварительные требования

- **JDK 11+**
- Каталог `lib/` должен содержать все необходимые JAR-файлы (в т.ч. Log4j 2)

---

### 🔨 Сборка

Из корневого каталога проекта выполните:

```bash
javac -d build -cp "lib/*" $(find src -name "*.java")
```

> Скомпилированные `.class` файлы будут помещены в папку `build/`

---

### ▶️ Запуск сервера

```bash
java -cp "build:lib/*" server.ServerMain
```

> Сервер начнёт слушать входящие TCP-подключения и логировать события в консоль.

---

### ▶️ Запуск клиента

```bash
java -cp "build:lib/*" client.ClientMain
```

> Клиент подключится к серверу и откроет интерактивный консольный интерфейс для ввода команд.

---
